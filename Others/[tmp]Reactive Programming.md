# Reactive Programming

reactive programming is a declarative programming paradigm concerned with data streams and the propagation of change. -Wikipedia

## 반응형 프로그래밍이란?

반응형 프로그래밍이란, **데이터의 흐름**과 변경사항의 전파에 중점을 둔 **선언적 프로그래밍** 패러다임이다. 

> **프로그래밍 페러다임**은 프로그래머에게 프로그래밍의 관점을 갖게 해주고 결정하는 역할을 한다. 좋은 프로그래밍 패러다임은 보다 좋은 프로그램을 만들 수 있는 방법과 시각을 제공해준다. 우리가 다양한 프로그래밍 패러다임을 알고 있다면 프로그램을 이해하는 데 훨씬 더 도움을 받을 수 있게 된다. 객체지향 프로그래밍이나 함수형 프로그래밍을 예로 들 수 있다.

## 웹에서의 반응형 프로그래밍

웹 개발에서 반응형 프로그래밍이 중요해진 계기는 '웹 프레임워크로의 전환'이다. DOM을 쉽게 조작할 수 있도록 도와주는 jQuery 에서 DOM을 알아서 렌더링해주는 웹 프레임워크로 패러다임이 전환되었다. 

여기서 웹 프레임워크의 본질을 살펴보자.

```
Change Detection + Binding + Template + Auto Render 
= 값이 변경되었을 때 템플릿에 선언해둔 대로 알아서 렌더링을 해준다.
= Reactive = 반응형 
```

웹 프레임워크의 본질과 반응형 프로그래밍의 정의가 유사한 것을 알 수 있다. 따라서 반응형 프로그래밍의 패러다임은 데이터의 변경을 감지하고 선언적으로 프로그래밍을 하는 방법을 통해 View를 업데이트 하는 방식으로 발전하기 시작한다. 

### 선언적 프로그래밍 

- **선언적 프로그래밍**이란 무엇을 해야할지 따로 약속(표현)을 만들어 기술하게 하고, 언제 어떻게 동작하는지 내부에서 처리하는 방식의 프로그래밍 기법이다. 
- 보통 명령형 프로그래밍에 비해 가독성, 재사용성, 유지보수성면에서 좋다. 

### 변경사항의 전파(pull -> push)

- **pull** : 관련 데이터가 필요한 시점에 데이터를 요청하는 방식
- **push** : 값이 변경될 때마다 변경된 값을 전파해서 사용하는 방식 (리액트의 `setState` 를 생각해보자.)
- DOM render를 해야 하는 시점에 필요한 데이터를 모두 불러와서 화면을 출력하는 **Pull 방식**에서, 템플릿이 미리 선언이 되어 있는 구조에 값이 변화할 때마다 템플릿으로 데이터를 전달하는 **Push 방식**으로 설계가 되도록 변화되었다. 

반응형 프로그래밍 패러다임은 뷰를 넘어 비즈니스 로직을 포함한 모든 스크립트에서 사용할 수 있도록 개념이 확장되었다. Redux 등을 이용한 상태 관리 역시 데이터의 변경을 감지하고 변경을 전파 후 선언적으로 값을 만들어낸다는 점에서 반응형 프로그래밍 패러다임에 속하게 된다.
